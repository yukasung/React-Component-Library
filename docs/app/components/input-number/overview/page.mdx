# InputNumber

InputNumber คือ input สำหรับกรอกตัวเลขโดยเฉพาะ ป้องกันไม่ให้ผู้ใช้กรอกข้อมูลที่ไม่ถูกต้องโดยไม่ตั้งใจ และจัดรูปแบบตัวเลขขณะแก้ไข ออกแบบ API ให้เป็นไปตามแนวทางของ React (value + onChange)

เอกสารหน้านี้จะทยอยเพิ่มเนื้อหาตามลำดับที่แต่ละ property ผ่านสถานะ Reviewed บน Linear เท่านั้น — ปัจจุบันมี value, placeholder, isRequired, min / max, step, format, showSpinButtons, repeatButtons, handleWheel, isReadOnly และ isDisabled ที่ผ่านแล้ว

### value

<PropertySignature name="value" type="number | null" />
<PropertySignature name="onChange" type="(value: number | null) => void" />

ค่าปัจจุบันของ control แบบ controlled — คู่กับ onChange เสมอ ถ้าไม่ระบุ value (ปล่อยให้เป็น undefined) component จะทำงานแบบ uncontrolled แทน โดยจัดการ state ภายในเอง เริ่มจาก defaultValue

value คือ**ข้อความที่แยกวิเคราะห์แล้ว** (parsed) เป็น number หรือ null ล้วน ๆ เสมอ ไม่มี comma คั่นหลักพัน สัญลักษณ์สกุลเงิน หรืออักขระอื่นปนอยู่เด็ดขาด — ไม่ว่าจะพิมพ์อะไรเข้าไปในช่อง สุดท้าย value ที่ได้จะเป็นตัวเลขบริสุทธิ์เสมอ (เช่น 1234.56 ไม่ใช่ "1,234.56") การจัดรูปแบบการแสดงผลด้วย comma คั่นหลักพัน สัญลักษณ์สกุลเงิน ฯลฯ เป็นคนละเรื่องกับ value โดยสิ้นเชิง ควบคุมแยกผ่าน property format ด้านล่าง

onChange ถูกเรียกเฉพาะตอนค่ามีการ commit จริง (blur, Enter, ปุ่ม spin, ปุ่มลูกศรบนคีย์บอร์ด) ไม่ใช่ทุกครั้งที่พิมพ์ — null ถือเป็นค่าที่ commit ได้ปกติ (ฟิลด์ว่างเปล่า) ต่างจาก draft ที่พิมพ์ไม่ครบ (เช่น "1." หรือ "-") ซึ่งจะไม่ถูกส่งเข้า onChange จนกว่าจะพิมพ์ให้เป็นตัวเลขที่สมบูรณ์

#### Live Example

<ValueDemo />

#### Example

```tsx
{/* Controlled */}
const [value, setValue] = useState<number | null>(10)
<InputNumber value={value} onChange={setValue} />

{/* Uncontrolled */}
<InputNumber defaultValue={10} onChange={(value) => console.log(value)} />
```

### placeholder

<PropertySignature name="placeholder" type="string" />

ข้อความ hint ที่แสดงเมื่อฟิลด์ว่างเปล่า เป็น native HTML attribute ส่งผ่านตรงเข้า input ภายในโดยไม่มีการแปลงใด ๆ

แสดงผลได้ก็ต่อเมื่อฟิลด์ว่างจริง ๆ (value เป็น null) เท่านั้น — ถ้าใช้คู่กับ isRequired (default: true) ฟิลด์จะไม่มีทางว่างเปล่าเลย เพราะ isRequired บังคับแสดงเป็น 0 เสมอ placeholder จึงไม่มีวันปรากฏ ต้องปิด isRequired ก่อนถึงจะเห็น placeholder จริง

#### Live Example

<PlaceholderDemo />

#### Example

```tsx
<InputNumber isRequired={false} defaultValue={null} placeholder="ไม่บังคับกรอก" />
```

### isRequired

<PropertySignature name="isRequired" type="boolean" />

เมื่อเป็น true ฟิลด์นี้ห้ามว่างเปล่า — ค่าที่เป็น null จะแสดงเป็น 0 เสมอ และหากผู้ใช้ลบข้อความจนหมด (ระหว่างพิมพ์ ไม่ต้องรอจน blur) ฟิลด์จะกระโดดกลับเป็น 0 ทันทีพร้อมเลือกตัวเลขนั้นไว้ ให้พิมพ์ทับได้เลย

ตั้งชื่อ prop เป็น isRequired โดยตั้งใจ แทนที่จะใช้ required แบบ native HTML ค่าเริ่มต้น (default): true

#### Live Example

<IsRequiredDemo />

#### Example

```tsx
{/* default: isRequired = true — ห้ามว่าง ไม่มีค่าจะแสดงเป็น "0" */}
<InputNumber defaultValue={null} />

{/* ปิด isRequired เพื่อให้เป็นฟิลด์ optional ปล่อยว่างได้ (commit เป็น null) */}
<InputNumber isRequired={false} defaultValue={null} placeholder="ไม่บังคับกรอก" />
```

### min / max

<PropertySignature name="min" type="number" />
<PropertySignature name="max" type="number" />

กำหนดค่าต่ำสุด/สูงสุดที่ยอมรับได้ การ clamp ค่าให้อยู่ในช่วงเกิดขึ้นเฉพาะตอน commit (blur, Enter, ปุ่ม spin, ปุ่มลูกศรบนคีย์บอร์ด) เท่านั้น ไม่ได้บล็อกระหว่างพิมพ์ ดังนั้นพิมพ์เลขที่เกินช่วงชั่วคราวระหว่างแก้ไขได้ตามปกติ ปุ่ม spin แต่ละปุ่มจะ disable เองเมื่อค่าที่ commit แล้วถึงขอบเขต (ปุ่มขึ้นที่ max, ปุ่มลงที่ min)

ไม่ระบุ min หรือ max หมายถึงไม่จำกัดด้านนั้น

#### Live Example

<MinMaxDemo />

#### Example

```tsx
<InputNumber min={0} max={100} defaultValue={50} />
```

### step

<PropertySignature name="step" type="number" />

จำนวนที่เพิ่ม/ลดต่อการกดปุ่ม spin หนึ่งครั้ง หรือกดปุ่มลูกศรขึ้น/ลงบนคีย์บอร์ดหนึ่งครั้ง ค่าเริ่มต้น (default): 1

ถ้า step มีทศนิยม (เช่น 0.1) และไม่ได้ระบุ precision ไว้เอง ระบบจะคำนวณจำนวนทศนิยมที่แสดงจาก step ให้อัตโนมัติ เพื่อไม่ให้เกิดปัญหาเศษทศนิยมเพี้ยนจากการคำนวณ floating point ซ้ำ ๆ (เช่น 0.1 บวกกันสามครั้งไม่ได้ 0.3 พอดีถ้าไม่จัดการ)

step แยกออกจากการแสดง/ซ่อนปุ่ม spin อย่างชัดเจน — step ควบคุมแค่ "ขยับทีละเท่าไร" เท่านั้น การซ่อน/แสดงปุ่ม spin ควบคุมแยกผ่าน showSpinButtons และปุ่มลูกศรบนคีย์บอร์ดยังคงขยับค่าได้เสมอไม่ว่าจะซ่อนปุ่ม spin หรือไม่

#### Live Example

<StepDemo />

#### Example

```tsx
<InputNumber step={0.1} defaultValue={1} />
```

### format

<PropertySignature name="format" type="string" />

รูปแบบการแสดงผลตัวเลข เขียนแบบ .NET standard numeric format string คือตัวอักษร 1 ตัวบอกประเภท ตามด้วยตัวเลขจำนวนทศนิยม/ความกว้างที่เป็นตัวเลือก (เช่น "n2", "C", "p0") เมื่อระบุ format จะแทนที่ precision เดิมทั้งหมด ทั้งการแสดงผลและจำนวนทศนิยมที่ใช้ปัดค่าตอน commit

รองรับครบทั้ง 9 ตัว:

| ตัวอักษร | ความหมาย | ตัวอย่าง (ค่า 1234.5) |
| --- | --- | --- |
| N | ตัวเลขมี comma คั่นหลักพัน | 1,234.50 |
| C | สกุลเงิน (นำหน้าด้วย $) | $1,234.50 |
| P | เปอร์เซ็นต์ (คูณค่าด้วย 100 แล้วต่อท้ายด้วย %) | 123,450.00% |
| F | ทศนิยมธรรมดา ไม่มี comma | 1234.50 |
| D | จำนวนเต็ม (ตัดทศนิยมทิ้ง) เติม 0 ด้านหน้าได้ | 1234 |
| E | สัญกรณ์วิทยาศาสตร์ | 1.234500E+003 |
| G | เลือกรูปแบบกะทัดรัดที่สุดระหว่างทศนิยมกับวิทยาศาสตร์ให้เอง | 1234.5 |
| X | เลขฐาน 16 (ตัดทศนิยมทิ้งก่อน) | 4D2 |
| R | ค่าที่แปลงกลับเป็นตัวเลขเดิมได้เป๊ะที่สุด | 1234.5 |

ตัวเลขที่ต่อจากตัวอักษร (เช่น "n2") กำหนดจำนวนทศนิยมที่แสดง — ยกเว้น D และ X ที่ความหมายเป็น "จำนวนหลักขั้นต่ำ เติม 0 ด้านหน้า" แทน ไม่ใช่ทศนิยม ถ้าไม่ระบุตัวเลขจะใช้ค่า default ของตัวอักษรนั้น (ส่วนใหญ่คือ 2 ตำแหน่ง) ตัวอักษรพิมพ์เล็ก/ใหญ่มีผลกับตัวพิมพ์ของผลลัพธ์ในบาง specifier (เช่น E/e, X/x) แต่ไม่มีผลกับความหมาย

ค่าติดลบแสดงด้วยเครื่องหมายลบนำหน้าเสมอในทุก specifier รวมถึงสกุลเงิน (ไม่ใช้วงเล็บแบบ .NET default เพื่อให้คำนวณตำแหน่ง cursor ตอนพิมพ์ได้ถูกต้อง)

การจัดรูปแบบเกิดขึ้น**ระหว่างพิมพ์จริง** ไม่ใช่แค่ตอน commit — พิมพ์ตัวเลขแล้วจะเห็น comma/สัญลักษณ์ปรากฏทันที ปุ่มลัดเครื่องหมายลบ/บวก (สลับ/ลบเครื่องหมาย) จะปิดใช้งานเมื่อ format ทำงานอยู่ เพราะรูปแบบตกแต่งของแต่ละ specifier ไม่มีตำแหน่งที่เป็นสากลพอจะรองรับได้ทุกกรณี ให้พิมพ์เครื่องหมายลบ/บวกตามปกติแทน ส่วนปุ่มจุดทศนิยมและ Backspace ยังทำงานตามปกติ (ปุ่มจุดทศนิยมกระโดดไปหลังจุดทศนิยมเดิมแทนการแทรกจุดที่สอง, Backspace ข้ามจุดทศนิยมแทนการลบทิ้งเมื่อ cursor อยู่ติดหลังจุด)

### showSpinButtons

<PropertySignature name="showSpinButtons" type="boolean" />

กำหนดว่าจะแสดงปุ่ม −/+ สำหรับเพิ่ม/ลดค่า (spin buttons) ขนาบสองข้างของฟิลด์หรือไม่ ค่าเริ่มต้น (default): true

เมื่อเป็น false ปุ่ม spin จะไม่ถูก render เลย (ไม่ใช่แค่ซ่อนด้วย CSS) เหลือแค่กรอบของฟิลด์เพียงอย่างเดียว — ปุ่มลูกศรขึ้น/ลงบนคีย์บอร์ดยังคงขยับค่าได้ตามปกติไม่ว่าจะซ่อนปุ่ม spin หรือไม่ก็ตาม

ตั้งชื่อ prop เป็น showSpinButtons (ไม่ใช่ showSpinner) และไม่ผูกกับค่า step ใด ๆ เป็นพิเศษ — ควบคุมแยกอิสระจากกันโดยสมบูรณ์

#### Live Example

<ShowSpinButtonsDemo />

#### Example

```tsx
<InputNumber showSpinButtons={false} defaultValue={10} />
```

### repeatButtons

<PropertySignature name="repeatButtons" type="boolean" />

เมื่อกดค้าง จะมีดีเลย์เล็กน้อยก่อนเริ่มขยับซ้ำ (กันการขยับโดยไม่ตั้งใจจากการคลิกสั้น ๆ) จากนั้นขยับต่อเนื่องเป็นจังหวะจนกว่าจะปล่อยเมาส์หรือค่าถึง min/max แล้วหยุดเอง — กดครั้งเดียวโดยไม่ค้างยังคงขยับ 1 step ตามปกติเสมอไม่ว่าตั้งค่านี้เป็นอะไร ไม่กระทบปุ่มลูกศรบนคีย์บอร์ด (ไม่มีการ repeat เมื่อกดค้างที่คีย์บอร์ด)

### handleWheel

<PropertySignature name="handleWheel" type="boolean" />

กำหนดว่าจะให้เลื่อนล้อเมาส์ (mouse wheel) ตอน focus อยู่ที่ฟิลด์เพื่อเพิ่ม/ลดค่าตาม step ได้หรือไม่ ค่าเริ่มต้น (default): **false**

ปิดโดย default โดยตั้งใจ (ต่างจากพฤติกรรมทั่วไปที่มักเปิดไว้) เพราะการเลื่อนล้อเมาส์เปลี่ยนค่าโดยไม่ตั้งใจเป็นปัญหาที่พบบ่อย เช่น เลื่อนหน้าเว็บผ่าน input ที่บังเอิญ focus ค้างอยู่ ต้องเปิดใช้งานเองผ่าน prop นี้ และทำงานเฉพาะตอนฟิลด์ focus อยู่เท่านั้น ไม่ทำงานถ้า isDisabled หรือ isReadOnly

### isReadOnly

<PropertySignature name="isReadOnly" type="boolean" />

เมื่อเป็น true ผู้ใช้แก้ไขค่าไม่ได้ทั้งการพิมพ์ ปุ่ม spin และปุ่มลูกศรบนคีย์บอร์ด แต่ยังคง focus, เลือกข้อความ และคัดลอกค่าได้ตามปกติ — ต่างจาก isDisabled ที่ปิดกั้นการโต้ตอบทั้งหมดและแสดงผลจางลง isReadOnly จึงมีสไตล์ที่แยกจาก isDisabled โดยเฉพาะเพื่อไม่ให้ดูเหมือนใช้งานไม่ได้ ค่าเริ่มต้น (default): false

### isDisabled

<PropertySignature name="isDisabled" type="boolean" />

เมื่อเป็น true ปิดการใช้งานทั้งหมด — พิมพ์ไม่ได้ ปุ่ม spin และปุ่มลูกศรบนคีย์บอร์ดใช้ไม่ได้ ทั้งยัง focus ไม่ได้ด้วย (ต่างจาก isReadOnly ที่ยัง focus/เลือกข้อความได้) แสดงผลจางลงชัดเจน (opacity ต่ำ พื้นหลังเทา cursor เป็น not-allowed) เพื่อสื่อว่าใช้งานไม่ได้ ค่าเริ่มต้น (default): false

ตั้งชื่อ prop เป็น isDisabled โดยตั้งใจ แทนที่จะใช้ disabled แบบ native HTML ค่าที่ส่งเข้าไปจะแปลงเป็น native disabled attribute บน input และปุ่ม spin ทั้งสองให้เองภายใน

## Accessibility

InputNumber ทำหน้าที่เป็น spinbutton (มีปุ่มเพิ่ม/ลดค่าและปุ่มลูกศรบนคีย์บอร์ด) จึงตั้ง role และ aria-value* attribute ให้เองอัตโนมัติตาม ARIA spinbutton pattern มาตรฐาน โดยไม่ต้องตั้งค่าอะไรเพิ่ม:

- role เป็น spinbutton (แทน textbox ปกติ) — screen reader จะประกาศว่าเป็นปุ่มปรับค่า ไม่ใช่แค่ช่องพิมพ์ข้อความ
- aria-valuenow คือค่าตัวเลขดิบปัจจุบัน (ไม่มี comma หรือสัญลักษณ์ใด ๆ)
- aria-valuetext คือข้อความที่แสดงจริงในช่อง (รวม format ถ้ามี เช่น "$1,234.50") ให้ screen reader อ่านแบบเดียวกับที่ตาเห็น
- aria-valuemin / aria-valuemax มาจาก min / max โดยตรงถ้ามีการตั้งค่าไว้

aria-valuenow และ aria-valuetext อัปเดตตามสิ่งที่พิมพ์อยู่แบบสด ๆ ทันทีที่พิมพ์ (ไม่ต้องรอ blur/Enter) ตรงกับสิ่งที่แสดงบนหน้าจอเสมอ แม้จะยังไม่ commit เป็นค่าจริงก็ตาม — ต่างจาก value/onChange ที่ยังทำงานตาม commit model ตามปกติ (ดูหัวข้อ value) ถ้าสิ่งที่พิมพ์ยังไม่ใช่ตัวเลขที่สมบูรณ์ (เช่น พิมพ์แค่เครื่องหมายลบ) จะไม่ใส่ aria-valuenow ให้ชั่วคราวจนกว่าจะพิมพ์ต่อจนเป็นตัวเลขได้

เมื่อยังไม่มีค่า (value เป็น null และไม่ได้บังคับ isRequired) จะไม่ใส่ aria-valuenow/aria-valuetext เลย ตรงตามที่ ARIA spec แนะนำสำหรับ spinbutton ที่ยังไม่มีค่า

นอกจากนี้ prop hint (ข้อความคำอธิบายที่แสดงใต้ input) จะเชื่อมกับ input ผ่าน aria-describedby ให้อัตโนมัติเสมอ โดยสร้าง id ให้เองและ merge เข้ากับ aria-describedby ที่ส่งมาเองด้วย ถ้ามี
